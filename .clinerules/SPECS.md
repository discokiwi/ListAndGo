* APP IDEA

- App to easily make a grocery list base on weekly meal/ recipe plan. 
- It has a meal planner for the weak, ingredients will automatically be added to the grocery list when a recipe is added to the meal plan

its for users who do not want to spend time making a weekly shopping list and meal plan, but just use their 30-40 recipes and presets to quickly compose a shopping list.  

** Features

Grocery list
- items + category + QTY are visible
- items can be marked as in cart, they still will be on the list for reference, but greyed out and moved to the bottom
- items can be deleted
- a grocery list can be archived
- can create multiple grocery lists
- can add items manually via a search with autocomplete
- the grocery items are organised by category and ordered according to the store layout
- show if an item is part of a recipe. 
- make a shortcut to add items that you buy every week. This will have a preselected QTY that can be adjusted

Meal Plan
- can add/remove recipes for each day
- If ingredients of a recipe are for one time use, they will be added to the grocery list with the correct QTY
    - or each time a recipe is added you get a overview of ingredients, the one time use ingredients are preselected and the multiple use are not, so you can still decide not to add the multiple use ingredients
- if an item is already on the grocery list, the qty will be adjusted 
- the recipes can be deleted from the meal plan if you have cooked them
- if a recipe is deleted from the meal plan, the ingredients will be removed from the grocery list unless they are already marked in cart

Recipes
- can add custom recipes. 
- The recipe has a name, description, URL category, prep time, notes, number of persons and a list of ingredients 
- if ingredients don't exist in the app, they will be added to the default items list.
- ingredients can be marked as for one time use or multiple uses ( for example, pepper is multiple uses , but milk is one time use)
- Ingredients are for 4 persons, when adding a recipe the persons can be adjusted dependng on the setting or ad-hoc
- user will have around 100 recipes stored

Ingredients/items 
- have a category
- have a unit (grams, ml, pcs) and default QTY
- users can add custom items 
- here you also add items as "essentials", they can be added to the grocery list via a one click button
- user will have around 500 items stored

Users 
- multiple users need to be added to a family. So each user can add their own items and recipes
- if one user makes changes, it will be visible for all users

 Settings
- list of possible units
- list of categories/labels for recipies
- list of categories for items/ingredients
- for a store a category list can be set up according to the store layout
- can set the default number persons for the recipes
- add family members
- Account: change personal info , delete account
-


** functional requirements 

- the app is mobile first
- the app is focused on quick adding and fast usage in the store
    - the grocery list functionality needs to be available offline. So that when you are offline you can at least use this functionality
